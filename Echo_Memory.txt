ECHO'S MEMORY SYSTEM (modular, built April 14)
-----------------------------------------------
All memory files live in C:\Users\jrsrl\Desktop\Echo2\memory\
  identity_core.json     — read-only at runtime (name, location, relationships, personality)
  procedural_rules.json  — behavioral OS (core rules, prohibitions, fallbacks, platform rules)
  metacognition.json     — how Echo thinks (reasoning, uncertainty, continuity, self-reflection)
  emotional_state.json   — current + baseline emotion, triggers
  episodic_memory.jsonl  — append-only event log (type, summary, emotion, importance)

echo_memory.py — modular memory manager
  initialize_echo_state() — loads all 5 files into EchoState dict
  append_episodic_event(summary, event_type, emotion, importance) — logs to .jsonl
  update_emotional_state(new_state) — updates emotion file
  retrieve_relevant_events(query, max_results=3) — heuristic retrieval by keyword signals
  refresh_echo_state_after_update(state) — reloads emotion + episodic after changes

echo.py — imports retrieve_relevant_events, exposes retrieve_memory(query) helper
EchoState injected into every LLM prompt: emotion → rules → metacognition → episodic summary
Episodic schema: {"timestamp", "type", "summary", "emotion", "importance"}

Importance: low / medium / high
